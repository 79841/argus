Argus는 AI 코딩 에이전트(Claude Code, Codex CLI, Gemini CLI)를 위한 개인용 모니터링 대시보드입니다. 에이전트에서 OpenTelemetry 데이터를 수집하고 사용량, 비용, 효율성을 통합 로컬 대시보드에서 시각화합니다.

> **인증이 필요 없습니다.** Argus는 로컬 머신에서만 실행됩니다.

## 공통 UI 요소

### 사이드바 내비게이션

왼쪽 사이드바에서 모든 페이지에 접근할 수 있습니다. 각 페이지는 아이콘과 레이블로 표시됩니다.

### FilterBar

대부분의 페이지 상단에 표시되는 필터 바입니다:

- **에이전트 필터** -- "All", "Claude", "Codex", "Gemini" 중 선택하여 특정 에이전트 데이터만 표시합니다.
- **프로젝트 필터** -- 특정 프로젝트 데이터만 표시합니다 (`OTEL_RESOURCE_ATTRIBUTES`에서 파생).
- **날짜 범위 선택** -- 표시할 데이터의 날짜 범위를 선택합니다.

### 하단 바

모든 페이지 하단에 표시되는 고정 푸터입니다:

- **에이전트 상태** -- 에이전트별 초록/노랑/회색 점으로 마지막 데이터 수신 시간을 표시합니다 (< 1시간 / < 24시간 / 그 이전).
- **활성 세션** -- 현재 실행 중인 세션의 에이전트 이름, 모델, 실행 비용을 표시합니다.
- **예산 진행률** -- 일일 비용 한도가 설정된 경우 에이전트별 미니 진행률 바를 표시합니다.
- **누적 합계** -- 모든 에이전트의 누적 비용과 토큰 수를 표시합니다.

### 테마와 언어

- **다크 / 라이트 / 시스템** 테마 모드.
- **한국어 / 영어** 언어 전환.
- **에이전트 테마** -- 선호하는 에이전트(Claude, Codex, Gemini)에 맞는 주 강조 색상을 선택합니다.

## 1. Overview (/)

랜딩 페이지로 모든 에이전트의 오늘 활동 스냅샷을 제공합니다.

### KPI 카드 (4개)

| 카드 | 설명 |
|------|------|
| **오늘 비용** | 오늘 모든 에이전트에서 발생한 총 비용. 어제 대비 변화율을 표시합니다 (감소가 양수). |
| **세션** | 오늘 시작된 세션 수. 어제 대비 변화율을 표시합니다. |
| **요청** | 오늘 발생한 총 API 요청 수. 어제 대비 변화율을 표시합니다. |
| **캐시 히트율** | 오늘의 캐시 읽기 토큰 대 총 입력 토큰 비율. 어제 대비 변화를 표시합니다. |

### 사용량 히트맵

GitHub 스타일의 기여 히트맵으로 최근 16주(112일)간의 일별 활동을 표시합니다. 각 셀은 하루를 나타내며 색상 강도는 사용량에 비례합니다.

### 오늘의 에이전트별 현황

각 에이전트의 오늘 활동을 보여주는 요약 카드입니다:

- 색상 점이 있는 에이전트 이름
- 오늘의 비용
- 요청 수
- 마지막 활동 이후 시간 (예: "5분 전")

### 최근 세션

모든 에이전트의 최근 5개 세션입니다. 각 항목은 다음을 표시합니다:

- 에이전트 점과 이름
- 프로젝트 이름 (있는 경우)
- 사용된 모델
- 비용과 소요 시간

세션을 클릭하면 해당 세션이 선택된 Sessions 페이지로 이동합니다.

### 팁

- Overview 페이지를 매일 확인하여 비정상적인 지출을 발견하세요.
- 히트맵은 주 단위, 월 단위 사용 패턴을 파악하는 데 도움이 됩니다.
- KPI 카드의 변화율로 오늘과 어제를 빠르게 비교할 수 있습니다.

## 2. Sessions (/sessions)

개별 에이전트 세션을 탐색하기 위한 마스터-디테일 뷰입니다.

### 필터

- **에이전트 필터** -- 특정 에이전트로 범위를 제한합니다.
- **프로젝트 필터** -- 특정 프로젝트로 범위를 제한합니다.
- **날짜 범위 선택** -- 기본값: 최근 7일.
- **검색** -- 세션 ID, 프로젝트 이름, 모델, 에이전트 유형에 대한 자유 텍스트 검색.
- **정렬** -- 최신순, 비용순, 토큰순으로 세션을 정렬합니다.

### 세션 목록 (왼쪽 패널, 35%)

스크롤 가능한 세션 목록입니다. 각 항목은 다음을 표시합니다:

- 에이전트 색상 점
- 모델 배지 (축약된 모델 이름)
- 총 세션 비용 (강조)
- 프로젝트 이름과 소요 시간
- 토큰 수, 캐시 히트율, 상대 시간

헤더에는 총 세션 수와 합산 비용이 표시됩니다.

세션을 클릭하면 오른쪽 패널에 상세 정보가 로드됩니다. 마우스를 올리면 전용 페이지로 이동하는 외부 링크 아이콘이 표시됩니다 (구현된 경우).

### 세션 상세 (오른쪽 패널, 65%)

세션을 선택하면 상세 패널에 다음이 표시됩니다:

#### 헤더
- 에이전트 배지와 모델 배지
- 축약된 세션 ID

#### 요약 그리드 (6개 지표)

| 지표 | 설명 |
|------|------|
| **비용** | 이 세션의 모든 API 요청 총 비용 |
| **입력** | 총 입력 토큰 |
| **출력** | 총 출력 토큰 |
| **캐시** | 캐시 읽기 토큰과 히트율 |
| **소요 시간** | 세션의 실제 경과 시간 |
| **요청 / 도구** | API 요청과 도구 호출 수 |

#### 모델별 비용 분석 차트

세션에서 사용된 모델별 비용 분석을 시각적으로 보여줍니다 (`SessionModelCostChart` 컴포넌트).

#### 이벤트 타임라인

탭을 통해 두 가지 보기 모드를 제공합니다:

- **목록** -- 프롬프트별로 그룹화된 이벤트. 각 프롬프트 그룹은 접기/펼치기가 가능하며 다음을 표시합니다:
  - 프롬프트 번호, 시작 시간, 이벤트 수, 비용
  - 펼치면 색상 코드가 있는 개별 이벤트를 나열합니다:
    - 파랑: API 요청 (모델, 입력/출력 토큰, 캐시 토큰, 비용)
    - 초록: 도구 결과 (도구 이름, 성공/실패 상태)
    - 보라: 사용자 프롬프트
    - 빨강: API 오류 또는 실패한 도구
  - 각 이벤트에 타임스탬프와 소요 시간 표시

- **워터폴** -- 이벤트의 시각적 워터폴/타임라인 차트 (`TraceWaterfall` 컴포넌트)로 이벤트가 시간상 어떻게 겹치는지 보여줍니다.

### 팁

- "비용"으로 정렬하여 가장 비싼 세션을 빠르게 찾으세요.
- 검색에 모델 이름을 입력하세요 (예: "opus" 또는 "sonnet").
- 각 세션 카드의 캐시 히트율로 캐싱 개선이 필요한 세션을 파악하세요.
- 목록 뷰에서 프롬프트 그룹을 펼쳐 어떤 API 호출이 비용이 높았는지 추적하세요.

## 3. Usage (/usage)

비용, 토큰, 모델, 효율성, 설정 영향을 다루는 5개 탭의 심층 분석 페이지입니다.

### 필터

- **에이전트 필터** -- 특정 에이전트로 범위를 제한합니다.
- **프로젝트 필터** -- 특정 프로젝트로 범위를 제한합니다.
- **날짜 범위 선택** -- 기본값: 최근 30일.

### 탭: Cost

#### KPI 카드 (3개)

| 카드 | 설명 |
|------|------|
| **총 비용** | 선택한 기간의 모든 비용 합계. 이전 동일 기간 대비 변화를 표시합니다. |
| **일 평균** | 총 비용을 기간 일수로 나눈 값. |
| **요청** | 기간 내 총 API 요청 수. |

#### 차트

- **일별 비용 추이** -- 에이전트별(Claude / Codex / Gemini) 일별 비용을 누적 영역 차트로 표시합니다.
- **에이전트별 비용** -- 에이전트별 총 비용을 가로 막대 차트로 비교합니다.
- **프로젝트별 비용** -- 프로젝트별 비용을 가로 막대 차트로 표시합니다 (상위 8개).

### 탭: Tokens

#### KPI 카드 (3개)

| 카드 | 설명 |
|------|------|
| **총 토큰** | 입력 + 출력 + 캐시 읽기 토큰의 합계. |
| **입력 / 출력 비율** | 입력 + 출력 대비 입력 토큰의 비율. |
| **캐시 절감** | 총 캐시 읽기 토큰 (처리 대신 캐시에서 제공된 토큰). |

#### 차트

- **일별 토큰 사용량** -- 일별 입력, 출력, 캐시 읽기 토큰을 누적 막대 차트로 표시합니다.
- **에이전트별 토큰 분포** -- 에이전트별 토큰 분석을 가로 누적 막대 차트로 표시합니다.

### 탭: Models

#### 모델 사용량 테이블

사용된 각 모델을 나열하는 테이블입니다:

| 컬럼 | 설명 |
|------|------|
| Model | 전체 모델 ID |
| Agent | 이 모델을 사용하는 에이전트 |
| Reqs | API 요청 수 |
| Cost | 총 비용 |
| Avg/req | 요청당 평균 비용 |

#### 차트

- **모델별 비용** -- 모델별 비용 점유율을 파이 차트로 표시합니다.
- **모델별 요청** -- 모델별 요청 수를 가로 막대 차트로 표시합니다 (상위 8개).

### 탭: Efficiency

#### KPI 카드 (3개)

| 카드 | 설명 |
|------|------|
| **캐시 히트율** | 모든 에이전트의 cache_read / (cache_read + input). |
| **평균 응답 시간** | 평균 API 요청 소요 시간 (초 단위). |
| **효율성 점수** | 캐시율, 토큰 효율성, 속도를 종합한 점수 (0-100). |

#### 차트

- **효율성 추이** -- 에이전트별 효율성 점수의 시간별 추이를 라인 차트로 표시합니다.
- **에이전트별 효율성** -- 에이전트별 캐시율, 토큰 효율성, 평균 속도, 종합 점수를 비교하는 테이블입니다.

### 탭: Impact

설정 변경 이력과 사용량에 대한 잠재적 영향을 표시합니다.

- **범위 필터** -- "All", "Project", "User" 범위를 전환하여 설정 변경을 필터링합니다.
- **설정 타임라인** -- 최근 90일간 감지된 설정 파일 변경의 타임라인 시각화로 `ConfigTimeline` 컴포넌트를 사용합니다.

### 팁

- Cost 탭의 일별 추이로 지출 급증을 발견하세요.
- Efficiency 탭을 정기적으로 확인하여 캐시 히트율이 건강한지 확인하세요 (50% 이상 권장).
- Models 탭으로 비싼 모델을 파악하세요 -- 일상적인 작업에는 더 저렴한 모델로 전환을 고려하세요.
- Impact 탭은 설정 변경 시점을 보여주어 비용/효율성 변화와의 상관관계를 파악하는 데 도움이 됩니다.

## 4. Tools (/tools)

AI 코딩 에이전트의 도구 사용 패턴을 분석합니다.

### 필터

- **에이전트 필터** -- 특정 에이전트로 범위를 제한합니다.
- **날짜 범위** -- 빠른 선택 버튼: 7일, 14일, 30일, 90일.

### KPI 카드 (4개)

| 카드 | 설명 |
|------|------|
| **총 호출** | 선택 기간의 총 도구 호출 수. |
| **성공률** | 성공한 도구 호출의 비율. |
| **평균 소요 시간** | 도구 호출당 평균 실행 시간. |
| **고유 도구** | 사용된 고유 도구 수. |

### 탭: Overview

- **상위 도구** -- 상위 15개 가장 많이 사용된 도구를 호출 수와 함께 가로 막대 차트로 표시합니다. 툴팁에 성공/실패 분석과 평균 소요 시간이 표시됩니다.
- **카테고리 분포** -- 카테고리별(Built-in, Orchestration, MCP) 도구 사용량을 트리맵으로 시각화합니다. 셀 크기는 호출 수에 비례합니다.

### 탭: Details

- **도구 상세 테이블** -- 카테고리, 에이전트, 호출 수, 성공/실패 수, 성공률, 평균 소요 시간을 포함한 모든 도구의 종합 테이블입니다.
- **개별 도구 테이블** -- 스킬, 에이전트, MCP 도구의 호출별 상세 기록입니다.
- **등록된 도구 카드** -- 사용 가능하도록 등록된 도구를 표시합니다.

### 탭: Trends

- **일별 도구 추이** -- 상위 8개 도구의 일별 사용량을 누적 영역 차트로 표시합니다.
- **일별 합계** -- 일별 총 도구 호출 수를 라인 차트로 표시합니다.

### 팁

- 특정 도구의 성공률이 낮으면 설정 문제나 도구 버그를 나타낼 수 있습니다.
- Trends 탭으로 도구 사용량이 시간에 따라 증가하는지 확인하세요 -- 비용 증가를 예측할 수 있습니다.
- 카테고리 분포 트리맵으로 빌트인 도구, 오케스트레이션 도구, MCP 통합 간의 균형을 파악하세요.

## 5. Projects (/projects)

여러 프로젝트(리포지토리/워크스페이스) 간 비용과 활동을 비교합니다.

### KPI 카드 (3개)

| 카드 | 설명 |
|------|------|
| **총 프로젝트** | 활동이 기록된 고유 프로젝트 수. |
| **총 비용** | 모든 프로젝트의 합산 비용. |
| **가장 활발한 프로젝트** | 세션 수가 가장 많은 프로젝트. |

### 비용 비교 차트

최대 15개 프로젝트의 비용을 비교하는 가로 막대 차트입니다. 각 막대는 시각적 구분을 위해 색상으로 구분됩니다.

### 프로젝트 테이블

모든 프로젝트를 나열하는 클릭 가능한 테이블입니다:

| 컬럼 | 설명 |
|------|------|
| Project | 프로젝트 이름 (클릭하면 상세 보기) |
| Sessions | 세션 수 |
| Requests | API 요청 수 |
| Cost | 총 비용 |
| Top Model | 가장 많이 사용된 모델 |
| Last Active | 마지막 활동 날짜 |

행을 클릭하면 프로젝트 상세 페이지로 이동합니다.

### 팁

- 프로젝트를 비교하여 어떤 코드베이스가 가장 많은 AI 리소스를 소비하는지 파악하세요.
- "Top Model" 컬럼으로 각 프로젝트가 가장 비용 효율적인 모델을 사용하고 있는지 확인하세요.

## 6. Insights (/insights)

AI 기반 제안과 비용 분석으로 에이전트 사용을 최적화합니다.

### 필터

- **날짜 범위** -- 빠른 선택 버튼: 7일, 14일, 30일, 90일.

### 제안

사용 패턴을 분석하고 실행 가능한 권장 사항을 제공하는 지능형 제안 엔진입니다. 각 제안 카드에는 다음이 포함됩니다:

- **심각도** -- Critical (빨강), Warning (주황), Info (파랑)과 해당 아이콘.
- **카테고리 배지** -- 제안이 관련된 영역.
- **현재 지표** -- 해당 지표의 현재 값.
- **목표** -- 권장 목표 값.
- **조치** -- 수행할 구체적인 단계.

문제가 감지되지 않으면 초록색 "모두 양호" 메시지가 표시됩니다.

### KPI 카드 (3개)

| 카드 | 설명 |
|------|------|
| **상위 10 합계** | 가장 비싼 세션 상위 10개의 합산 비용. |
| **평균 세션 비용** | 상위 10개 세션의 평균 비용. |
| **주요 원인** | 고비용 세션의 가장 흔한 원인 (예: "고가 모델", "다수 도구 호출", "캐시 없음"). |

### 일일 예산 게이지

일일 비용 한도가 설정된 경우 (Settings에서) 게이지 섹션이 표시됩니다:

- 에이전트별 사용액 대 한도 진행률 바
- 예산 초과 시 빨간색 강조

### 고비용 세션 테이블

가장 비싼 세션 상위 10개의 테이블입니다:

| 컬럼 | 설명 |
|------|------|
| Agent | 세션을 실행한 에이전트 |
| Model | 사용된 모델 |
| Cost | 총 세션 비용 |
| Reqs | API 요청 수 |
| Tools | 도구 호출 수 |
| Causes | 세션이 비쌌던 이유를 설명하는 색상 배지 (예: "고가 모델", "다수 요청", "캐시 없음") |

### 모델 비용 효율성 테이블

모델 간 비용 효율성을 비교합니다:

| 컬럼 | 설명 |
|------|------|
| Model | 모델 이름 |
| Agent | 사용하는 에이전트 |
| Reqs | 요청 수 |
| Total Cost | 이 모델의 총 비용 |
| Avg/Req | 요청당 평균 비용 |
| Per 1K Tok | 1,000 토큰당 비용 |
| Avg Speed | 평균 응답 시간 |

### 팁

- Insights를 정기적으로(주 1회) 확인하여 비용 최적화 기회를 조기에 발견하세요.
- "Critical" 심각도 제안에 주의하세요 -- 일반적으로 상당한 절감 가능성을 나타냅니다.
- 고비용 세션 테이블의 "Causes" 컬럼으로 세션이 비싼 *이유*를 파악하세요.
- 모델 효율성 테이블로 워크플로우에 가장 적합한 가격 대비 성능 모델을 선택하세요.

## 7. Rules (/rules)

대시보드에서 직접 에이전트 설정 파일을 보고 편집합니다.

### 파일 트리 (왼쪽 패널, 35%)

다음 기준으로 구성된 계층적 파일 브라우저입니다:

1. **범위** -- "Project" (현재 프로젝트 디렉토리) 또는 "User" (전역 사용자 설정).
2. **에이전트** -- Claude, Codex, Gemini.
3. **파일** -- 컨텍스트 아이콘이 있는 개별 설정 파일:
   - `.mcp.json` -- MCP 플러그인 아이콘
   - `.json` / `.toml` -- 설정 기어 아이콘
   - `agents/` / `skills/` -- 폴더 아이콘
   - 기타 마크다운 파일 -- 문서 아이콘

접기/펼치기 그룹으로 범위와 에이전트를 확장/축소할 수 있습니다.

### 파일 뷰어/에디터 (오른쪽 패널, 65%)

파일을 선택하면:

#### 헤더
- 전체 파일 경로
- 범위 배지 (Project / User)
- **미리보기**와 **편집** 모드 전환 버튼

#### 미리보기 모드
- **마크다운 파일** -- 제목, 코드 블록, 테이블, 목록, 링크, 인라인 서식이 렌더링됩니다.
- **JSON 파일** -- 키, 문자열, 숫자, 불리언이 색상으로 구분된 구문 강조가 적용됩니다.
- **TOML 파일** -- 섹션 헤더, 키-값 쌍, 주석이 구문 강조됩니다.
- **기타 파일** -- 모노스페이스 글꼴의 일반 텍스트로 표시됩니다.

#### 편집 모드
- 파일 내용을 편집하는 전체 높이 텍스트 영역.
- 디스크에 변경사항을 저장하는 **저장** 버튼.
- 저장 후 성공 확인 메시지.

### 팁

- Rules를 사용하여 대시보드를 떠나지 않고 에이전트 설정을 빠르게 확인하고 조정하세요.
- 마크다운 파일의 미리보기 모드로 에이전트 지시사항과 스킬 정의를 쉽게 읽을 수 있습니다.
- 편집 모드에서 MCP 설정, 에이전트 프롬프트 등을 즉시 업데이트할 수 있습니다.

## 8. Settings (/settings)

Argus 대시보드 자체를 설정합니다. 왼쪽 사이드바를 통해 5개 섹션으로 구성됩니다.

### General

| 설정 | 설명 |
|------|------|
| **테마** | Light, Dark, System 모드를 선택합니다. |
| **에이전트 테마** | 주 강조 색상을 Claude (주황), Codex (초록), Gemini (파랑)으로 설정합니다. 대시보드 전체 색상 강조가 변경됩니다. |
| **언어** | 한국어와 영어를 전환합니다. |
| **자동 새로고침** | 대시보드 데이터 자동 새로고침을 30초, 1분, 5분 간격으로 설정하거나 끕니다. |

### Agents

| 설정 | 설명 |
|------|------|
| **비용 한도** | 에이전트별 일일 및 월간 비용 한도를 설정합니다. *(준비 중 -- 현재 비활성화.)* |
| **수집 상태** | 각 에이전트의 텔레메트리 전송 여부를 모니터링합니다. *(준비 중.)* |

### Pricing

| 설정 | 설명 |
|------|------|
| **LiteLLM 가격 동기화** | "Sync Now"를 클릭하여 LiteLLM 가격 데이터베이스에서 최신 토큰 가격 정보를 가져옵니다. 비용 계산이 현재 모델 가격을 반영하도록 합니다. |

### Setup

에이전트별 단계별 설정 가이드입니다:

- **Claude Code** -- OTLP 텔레메트리 환경 변수, 프로젝트 이름 설정, 도구 상세 로깅.
- **Codex** -- OTLP HTTP 익스포터를 위한 TOML 설정.
- **Gemini CLI** -- 텔레메트리 내보내기를 위한 JSON 또는 환경 변수 설정.
- **Dashboard** -- Argus 대시보드 설치 및 실행 방법.
- **지원 이벤트** -- 이벤트 유형 참조 목록 (`api_request`, `user_prompt`, `tool_result`, `tool_decision`, `api_error`).

### Data

| 설정 | 설명 |
|------|------|
| **내보내기** | CSV/JSON으로 데이터를 내보냅니다. *(준비 중.)* |
| **데이터 정리** | 오래된 데이터를 정리합니다. *(준비 중.)* |
| **DB 통계** | 데이터베이스 크기와 레코드 수를 확인합니다. *(준비 중.)* |

### 팁

- 처음 설정 후 Settings > Pricing에서 "Sync Now"를 클릭하여 정확한 비용 추적을 보장하세요.
- 작업 중 대시보드가 업데이트되길 원하면 자동 새로고침을 30초 또는 1분으로 설정하세요.
- Setup 섹션을 새 에이전트를 설정할 때 참고하세요 -- 복사-붙여넣기할 수 있는 정확한 명령어가 포함되어 있습니다.

## 빠른 시작 체크리스트

1. **대시보드 시작**: `cd dashboard && pnpm dev`
2. **에이전트 설정** (Settings > Setup에서 정확한 명령어 참조)
3. **가격 동기화**: Settings > Pricing > Sync Now 방문
4. **테스트 데이터 시드** (선택): `POST http://localhost:9845/api/seed`
5. **AI 에이전트로 코딩 시작** -- 데이터가 자동으로 전송됩니다
6. **Overview 확인**으로 오늘의 활동 확인
7. **Sessions 탐색**으로 개별 대화 상세 확인
8. **Insights 주간 리뷰**로 사용 최적화
