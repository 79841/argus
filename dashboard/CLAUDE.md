# Argus Dashboard

Next.js 기반 AI 코딩 에이전트 모니터링 대시보드. 인증 없이 로컬에서 실행한다.

## 기술 스택

- **프레임워크**: Next.js 15+ (App Router)
- **언어**: TypeScript (strict mode)
- **스타일**: Tailwind CSS + shadcn/ui
- **차트**: Recharts
- **DB**: SQLite (`better-sqlite3`, WAL 모드)
- **패키지 매니저**: pnpm

## 프로젝트 구조 (Feature-Based)

```
src/
├── app/                                # Next.js App Router — 라우팅만 담당
│   ├── layout.tsx                      # 루트 레이아웃
│   ├── {page}/page.tsx                # 각 페이지 → features/{name} import
│   ├── api/                           # API 라우트
│   │   └── {name}/route.ts
│   └── v1/                            # OTLP 호환 라우트
│
├── features/                           # ★ Feature 모듈 (도메인별)
│   └── {feature-name}/                 # 예: sessions, tools, usage ...
│       ├── components/                 # 페이지 전용 컴포넌트
│       ├── lib/                        # 페이지 전용 비즈니스 로직
│       ├── hooks/                      # 페이지 전용 훅
│       ├── __tests__/                  # 테스트 (co-location)
│       └── index.ts                    # public API (barrel export)
│
├── shared/                             # ★ 공유 모듈 (2개+ feature에서 사용)
│   ├── components/
│   │   ├── ui/                         # shadcn/ui 컴포넌트
│   │   └── {component}.tsx            # 공유 컴포넌트 (필터, 네비게이션 등)
│   ├── hooks/                          # 공유 훅
│   └── lib/                            # 공유 유틸리티
│       ├── db.ts                       # SQLite 클라이언트 + 스키마
│       ├── queries/                    # DB 쿼리 함수
│       └── {util}.ts                  # 포맷팅, 에이전트 정의 등
│
├── infra/                              # ★ 외부 시스템 연동
│   └── {module}.ts                    # git CLI, HTTP fetch 등 side-effect 모듈
│
├── middleware.ts
└── types/
```

### 구조 원칙

| 디렉토리 | 역할 | 배치 기준 |
|----------|------|----------|
| `app/` | 라우팅만 담당 | Next.js 규칙. 페이지는 feature를 import하여 렌더링 |
| `features/` | 도메인별 컴포넌트·로직·테스트 | 단일 페이지(또는 밀접한 페이지 그룹) 전용 |
| `shared/` | 공유 컴포넌트·훅·유틸 | 2개 이상 feature에서 사용 |
| `infra/` | 외부 시스템 연동 | git CLI, HTTP fetch 등 side-effect 모듈 |

### Feature 모듈 규칙

- 각 feature는 `components/`, `lib/`, `hooks/`, `__tests__/`, `index.ts`로 구성 (필요한 것만)
- `index.ts`에서 barrel export — 외부에서는 `@/features/{name}`으로 import
- feature 간 직접 import 금지 — 공유가 필요하면 `shared/`로 승격
- 테스트는 해당 모듈 옆에 co-location (`__tests__/` 디렉토리)

### SQLite 쿼리

```typescript
// src/lib/queries.ts 패턴
import { getDb } from './db'

export const getOverviewStats = async (agentType: string) => {
  const db = getDb()
  const filter = agentType !== 'all' ? 'AND agent_type = ?' : ''
  const params = agentType !== 'all' ? [agentType] : []
  return db.prepare(`SELECT ... FROM agent_logs WHERE ... ${filter}`).get(...params)
}
```

### 컴포넌트

- Arrow function + Named export
- Props는 `type` 키워드로 정의
- shadcn/ui 컴포넌트는 `src/shared/components/ui/`에 위치
- 차트: Recharts (`ResponsiveContainer`, `LineChart`, `BarChart`, `PieChart`)

### 에이전트 필터

모든 데이터 페이지에 에이전트 필터가 포함된다:
- `all`: 전체
- `codex`: Codex CLI
- `claude`: Claude Code
- `gemini`: Gemini CLI

쿼리 파라미터 `agent_type`으로 필터링한다.
