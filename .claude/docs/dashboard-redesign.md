# Argus 대시보드 페이지 재설계

## 1. 현재 문제점

| 문제 | 설명 |
|------|------|
| 페이지 중복 | Daily, Cost, Trends가 비슷한 차트를 다른 페이지에서 반복 |
| Overview 과부하 | KPI + 히트맵 + 파이차트 + 바차트 + 세션 목록을 한 페이지에 모두 |
| 드릴다운 부재 | Dashboard → Session 상세로 자연스럽게 이동하는 흐름 없음 |
| 네비게이션 불균형 | Monitoring(4) + Analysis(1)로 카테고리 불균형 |
| Efficiency 고립 | 효율성이 별도 페이지인데 실제로는 Usage의 일부 |

## 2. 참고 플랫폼

| 플랫폼 | 참고 요소 |
|--------|----------|
| Langfuse | Traces 중심 탐색, Sessions 그룹핑, Custom Dashboard 위젯 |
| Helicone | Requests 필터링, Properties 슬라이싱, 실시간 Dashboard |
| LangSmith | Projects 기반 조직, Monitoring 차트, Annotation Queues |
| OpenAI Usage | Cost/Activity 뷰 분리, 모델별 토큰 분석 |
| Anthropic Console | Dashboard 커맨드 센터, Usage 드릴다운, 프로젝트별 지출 한도 |
| Datadog APM | Service Map, Trace Explorer, Flame graph |
| Braintrust | Logs 필터링, Dashboards 메타데이터 슬라이싱 |

### 플랫폼 간 공통 패턴

| 패턴 | 해당 플랫폼 |
|------|------------|
| Overview/Dashboard (비용, 토큰, 요청 수 요약) | 전체 |
| Traces/Requests (개별 요청 상세) | Langfuse, Helicone, LangSmith, Datadog, Braintrust |
| Sessions (멀티턴 대화 그룹핑) | Langfuse, Helicone, LangSmith |
| Usage/Billing (API 사용량/비용) | OpenAI, Anthropic |
| Custom Dashboards (위젯 조합) | Langfuse, LangSmith, Datadog, Braintrust |

## 3. 새 페이지 구조 (10개 → 6개)

```
Dashboard      /              오늘 한눈에 — KPI, 활동, 실시간 상태
Sessions       /sessions      세션 탐색 — 리스트 + 상세 타임라인 (핵심 페이지)
Usage          /usage         비용·토큰·모델·효율성 통합 분석
Tools          /tools         도구 사용 패턴 분석
Rules          /rules         에이전트 지침·스킬·설정 파일 관리
Settings       /settings      앱 설정 + 초기 가이드
```

### 통합/제거 맵

| 기존 페이지 | 새 위치 | 이유 |
|------------|---------|------|
| Overview (`/`) | **Dashboard** | 이름 변경 + 구성 재설계 |
| Sessions (`/sessions`) | **Sessions** | 유지, 대폭 강화 |
| Daily (`/daily`) | **Usage** 탭으로 흡수 | Cost/Trends와 중복 |
| Cost (`/cost`) | **Usage** 탭으로 흡수 | Daily/Trends와 중복 |
| Trends (`/trends`) | **Usage** 탭으로 흡수 | Daily/Cost와 중복 |
| Efficiency (`/efficiency`) | **Usage** 탭으로 흡수 | 비용/토큰과 같은 맥락 |
| Tools (`/tools`) | **Tools** | 유지 |
| Config History (`/config-history`) | **Usage > Impact** 탭으로 흡수 | 변경 영향 분석 맥락 |
| Setup (`/setup`) | **Settings** 탭으로 흡수 | 소규모 기능 |
| Settings (`/settings`) | **Settings** | 유지, 탭 확장 |

> **신규 페이지**: Rules (`/rules`) — 에이전트 지침, 스킬, 에이전트 정의, MCP 설정 등 에이전트 동작을 정의하는 파일을 프로젝트/사용자 스코프별로 조회·편집. 기존 Settings > Config의 FileViewer를 독립 페이지로 승격.

### 네비게이션

```
┌──────────────┐
│  Dashboard   │  ← 기본
│  Sessions    │  ← 핵심 (Langfuse Traces 패턴)
│  Usage       │  ← 분석 (OpenAI/Anthropic Usage 패턴)
│  Tools       │
│  Rules       │  ← 신규 (Langfuse Prompts 패턴)
│──────────────│
│  Settings    │  ← 하단
└──────────────┘
```

카테고리 그룹핑 없이 **플랫 구조** 6개.

## 4. 전역 필터 변경

| 항목 | 현재 | 변경 |
|------|------|------|
| 위치 | TopBar (전 페이지 공통) | **각 페이지 내부**로 이동 |
| 이유 | Dashboard는 필터 불필요, Settings에도 불필요 | 페이지별로 필요한 필터만 표시 |
| 적용 | Sessions, Usage, Tools만 필터 표시 | |

## 5. 페이지별 상세 설계

### 5.1 Dashboard (`/`)

> 참고: Langfuse Dashboard, Anthropic Console Dashboard, Helicone Dashboard

**목적**: 오늘의 활동을 한눈에 파악. 이상 징후 빠르게 감지.

```
┌─────────────────────────────────────────────────────────┐
│  KPI 카드 (4개, 전일 대비 변화율 포함)                        │
│  ┌──────────┬──────────┬──────────┬──────────┐          │
│  │ 오늘 비용  │ 세션 수   │ 요청 수   │ 캐시율    │          │
│  │ $12.40   │ 8        │ 142      │ 67%      │          │
│  │ ▲ 23%    │ ▼ 12%    │ ▲ 5%     │ ▲ 3%     │          │
│  └──────────┴──────────┴──────────┴──────────┘          │
├─────────────────────────────────────────────────────────┤
│  활동 히트맵 (16주 × 7일, GitHub Contribution 스타일)       │
│  ░░▓▓██░░▓▓██░░▓▓██░░▓▓██░░▓▓██░░                      │
├──────────────────────────┬──────────────────────────────┤
│  에이전트별 오늘 요약 (3장)  │  최근 세션 (5개)              │
│  ┌────────────────────┐  │  ┌────────────────────────┐  │
│  │ Claude  $8.20  95건│  │  │ Claude · src/lib       │  │
│  │ Codex   $3.10  32건│  │  │   opus · $2.30 · 12m   │  │
│  │ Gemini  $1.10  15건│  │  │ Codex · dashboard      │  │
│  └────────────────────┘  │  │   gpt-4.1 · $0.80 · 5m │  │
│                          │  │ ...                      │  │
│                          │  │        → 전체보기 (링크)   │  │
│                          │  └────────────────────────┘  │
└──────────────────────────┴──────────────────────────────┘
```

| 영역 | 내용 | 데이터 |
|------|------|--------|
| KPI 카드 (4개) | 오늘 비용, 세션수, 요청수, 캐시 히트율 — 각각 전일 대비 변화율 | `/api/overview` |
| 활동 히트맵 | 16주 SVG 격자, 4모드(Agents/Sessions/Cost/Efficiency) | `/api/daily` |
| 에이전트 요약 | 에이전트별 오늘 비용 + 요청수 + 마지막 활동 시간 | `/api/overview` |
| 최근 세션 | 최근 5개 세션 (에이전트, 프로젝트, 모델, 비용, 소요시간). 클릭 시 Sessions 상세로 이동 | `/api/sessions?limit=5` |

**기존 대비 변경:**
- 모델 파이차트, 도구 바차트 제거 (각각 Usage, Tools로 이동)
- 전일 대비 변화율 추가
- 에이전트 요약 카드 신규
- 최근 세션 10→5개로 줄이되 클릭 드릴다운 추가

### 5.2 Sessions (`/sessions`) — 핵심 페이지

> 참고: Langfuse Traces, Helicone Requests, Datadog Trace Explorer

**목적**: 모든 세션을 탐색하고, 개별 세션의 이벤트 타임라인을 상세 분석.

```
┌────────────────────────────┬────────────────────────────────────┐
│  필터 바                    │                                    │
│  [에이전트▾] [프로젝트▾]     │                                    │
│  [날짜 범위] [검색...]       │                                    │
├────────────────────────────┤  세션 상세                           │
│  세션 리스트                 │                                    │
│ ┌────────────────────────┐ │  ┌──────────────────────────────┐  │
│ │ claude · opus          │ │  │ 세션 요약                      │  │
│ │  my-project · 12분 전   │ │  │ 비용: $2.30  토큰: 45.2K     │  │
│ │  $2.30 · 45.2K tok     │ │  │ 소요: 12m   캐시율: 72%       │  │
│ │  캐시율 72% · 12분      │ │  │ 요청: 8건   도구: 23회        │  │
│ ├────────────────────────┤ │  └──────────────────────────────┘  │
│ │ codex · gpt-4.1        │ │                                    │
│ │  dashboard · 1시간 전    │ │  이벤트 타임라인                    │
│ │  $0.80 · 12.1K tok     │ │  ┌──────────────────────────────┐  │
│ ├────────────────────────┤ │  │ 14:32:01 user_prompt          │  │
│ │ claude · sonnet        │ │  │          prompt_length: 245   │  │
│ │  argus · 2시간 전       │ │  │ 14:32:03 api_request         │  │
│ │  $1.10 · 28.5K tok     │ │  │          opus · 12.3K in      │  │
│ ├────────────────────────┤ │  │          2.1K out · $0.45     │  │
│ │ ...                    │ │  │ 14:32:08 tool_result          │  │
│ │                        │ │  │          Read · 성공 · 120ms  │  │
│ │  페이지네이션             │ │  │ 14:32:10 tool_result          │  │
│ └────────────────────────┘ │  │          Edit · 성공 · 85ms   │  │
│                            │  │ 14:32:15 api_request          │  │
│                            │  │          sonnet · 8.1K in     │  │
│                            │  │          1.8K out · $0.12     │  │
│                            │  │ ...                            │  │
│                            │  └──────────────────────────────┘  │
└────────────────────────────┴────────────────────────────────────┘
```

#### 좌측: 세션 리스트 (35%)

| 요소 | 설명 |
|------|------|
| 필터 바 | 에이전트, 프로젝트, 날짜 범위, 텍스트 검색 |
| 세션 카드 | 에이전트 아이콘, 모델, 프로젝트, 비용, 토큰, 캐시율, 소요시간, 상대시간 |
| 정렬 | 최신순(기본), 비용순, 토큰순 |
| 페이지네이션 | 무한 스크롤 또는 100건씩 |

#### 우측: 세션 상세 (65%)

| 영역 | 내용 |
|------|------|
| 세션 요약 | 총비용, 총토큰(input/output/cache), 소요시간, 캐시율, 요청수, 도구호출수 |
| 이벤트 타임라인 | 시간순 이벤트 (api_request, tool_result, user_prompt 등). 이벤트 타입별 아이콘/색상 |
| 빈 상태 | 세션 미선택 시 "세션을 선택하세요" 안내 |

**기존 대비 변경:**
- 세션 카드에 캐시율, 프로젝트 정보 추가
- 세션 요약 패널 신규 (상세 진입 시 상단에 KPI)
- 이벤트 타임라인에 이벤트 타입별 아이콘/색상 차별화
- 정렬 옵션 추가

### 5.3 Usage (`/usage`)

> 참고: OpenAI Usage, Anthropic Console Usage, Langfuse Dashboard

**목적**: 비용·토큰·모델·효율성을 하나의 페이지에서 탭으로 분석. 기존 Daily + Cost + Trends + Efficiency 통합.

```
┌─────────────────────────────────────────────────────────┐
│  [에이전트▾] [프로젝트▾] [날짜 범위]          필터 바       │
├─────────────────────────────────────────────────────────┤
│  [Cost] [Tokens] [Models] [Efficiency] [Impact]  탭 네비게이션  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│              (선택된 탭의 콘텐츠)                           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### Cost 탭

```
┌─────────────────────────────────────────────────────────┐
│  KPI: 기간 총비용 · 일평균 · 전기간 대비 변화율              │
├─────────────────────────────────────────────────────────┤
│  일별 비용 추이 (Stacked Area Chart)                      │
│  claude / codex / gemini 에이전트별 색상                   │
├───────────────────────────┬─────────────────────────────┤
│  에이전트별 비용 비교        │  프로젝트별 비용 비교          │
│  (수평 바 차트)             │  (수평 바 차트)               │
└───────────────────────────┴─────────────────────────────┘
```

| 영역 | 데이터 |
|------|--------|
| KPI 카드 | 기간 총비용, 일평균, 전기간 대비 변화율 |
| 일별 비용 추이 | Stacked Area Chart, 에이전트별 색상, 설정 변경 마커 |
| 에이전트별 비용 | 수평 바 차트 |
| 프로젝트별 비용 | 수평 바 차트 |

#### Tokens 탭

```
┌─────────────────────────────────────────────────────────┐
│  KPI: 총 토큰 · 입력/출력 비율 · 캐시 절감량                │
├─────────────────────────────────────────────────────────┤
│  일별 토큰 사용 (Stacked Bar Chart)                       │
│  input / output / cache_read / cache_creation            │
├───────────────────────────┬─────────────────────────────┤
│  에이전트별 토큰 분포        │  입력 vs 출력 비율 추이        │
└───────────────────────────┴─────────────────────────────┘
```

| 영역 | 데이터 |
|------|--------|
| KPI 카드 | 총 토큰, 입력/출력 비율, 캐시 절감량 |
| 일별 토큰 | Stacked Bar Chart (input/output/cache_read/cache_creation) |
| 에이전트별 토큰 | 분포 차트 |
| 입출력 비율 추이 | 라인 차트 |

#### Models 탭

```
┌─────────────────────────────────────────────────────────┐
│  모델 사용량 테이블                                       │
│  ┌─────────┬────────┬────────┬────────┬───────┐         │
│  │ Model   │ Reqs   │ Tokens │ Cost   │ Avg   │         │
│  ├─────────┼────────┼────────┼────────┼───────┤         │
│  │ opus    │ 142    │ 450K   │ $8.20  │ $0.06 │         │
│  │ sonnet  │ 89     │ 120K   │ $2.10  │ $0.02 │         │
│  │ gpt-4.1 │ 32    │ 85K    │ $3.10  │ $0.10 │         │
│  └─────────┴────────┴────────┴────────┴───────┘         │
├───────────────────────────┬─────────────────────────────┤
│  모델별 비용 도넛 차트       │  모델별 요청 추이 라인 차트    │
└───────────────────────────┴─────────────────────────────┘
```

| 영역 | 데이터 |
|------|--------|
| 모델 테이블 | 모델별 요청수, 토큰, 비용, 평균 비용 |
| 비용 도넛 | 모델별 비용 비율 |
| 요청 추이 | 모델별 일별 요청수 라인 차트 |

#### Impact 탭

> 기존 Config History 페이지의 분석 기능을 흡수. 설정 변경이 성능에 미친 영향을 추적.

```
┌─────────────────────────────────────────────────────────┐
│  스코프 필터: [All] [Project] [User]                      │
├─────────────────────────────────────────────────────────┤
│  설정 변경 타임라인 (Git diff 기반)                         │
│  ──●── 3/12 CLAUDE.md (project) ──●── 3/14 ~/.claude/settings.json (user) ── │
├─────────────────────────────────────────────────────────┤
│  선택한 변경의 전후 비교                                    │
│  ┌────────────────────┬────────────────────────────┐    │
│  │   Before (7일)      │    After (7일)              │    │
│  │   Avg Cost: $8.2    │    Avg Cost: $6.1  ↓25%    │    │
│  │   Cache Rate: 72%   │    Cache Rate: 85% ↑13%    │    │
│  │   Tool Fail: 5%     │    Tool Fail: 2%   ↓3%     │    │
│  └────────────────────┴────────────────────────────┘    │
├─────────────────────────────────────────────────────────┤
│  Diff 뷰어 (선택한 변경의 파일 diff, 읽기 전용)             │
└─────────────────────────────────────────────────────────┘
```

**설정 파일 스코프 분류:**

| 스코프 | 경로 예시 | 설명 |
|--------|----------|------|
| **Project** | `./CLAUDE.md`, `./.claude/settings.json`, `./codex.md`, `./GEMINI.md`, `./.mcp.json` | 프로젝트 디렉토리 내 설정. Git으로 추적 가능 |
| **User** | `~/.claude/settings.json`, `~/.codex/config.toml`, `~/.gemini/settings.json` | 사용자 홈 디렉토리 설정. 전역 적용, Git 추적 대상 아님 |

| 영역 | 데이터 |
|------|--------|
| 스코프 필터 | All / Project / User — 설정 파일의 적용 범위별 필터링 |
| 설정 변경 타임라인 | Git 커밋 기반(Project) + 파일 변경 감지(User). 스코프 배지 표시 |
| 전후 비교 | 변경 시점 전후 7일간 비용/캐시율/도구실패율 비교 |
| Diff 뷰어 | 선택한 변경의 파일 diff — **읽기 전용** (편집은 IDE에서) |

> **편집 기능 제외 사유**: Argus는 모니터링 도구이며, 설정 파일 편집은 IDE(VSCode, Cursor 등)의 역할이다.
> 프로젝트 레벨 파일은 Git으로 관리되고, 사용자 레벨 파일은 홈 디렉토리에 있으므로
> Argus에서 직접 편집하는 것은 범위를 벗어난다. Argus는 변경을 **감지하고 영향을 분석**하는 데 집중한다.

### 5.6 Settings (`/settings`)

> 기존 Settings + Setup 통합. 앱 자체의 설정만 담당

```
┌─────────────────────────────────────────────────────────┐
│  [General] [Agents] [Pricing] [Setup] [Data]            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│              (선택된 탭의 콘텐츠)                           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

| 탭 | 항목 | 설명 |
|------|------|------|
| **General** | 테마 | 라이트/다크/시스템 전환 |
| | 에이전트 테마 | Claude(Orange) / Codex(Green) / Gemini(Blue) 브랜드 테마 전환. Nav 액센트, 차트 강조, 카드 보더에 적용 |
| | 자동 갱신 | polling 주기 설정 (30초/1분/5분/끔) |
| **Agents** | 비용 한도 | 에이전트별 일일/월별 비용 한도 설정 |
| | 수집 상태 | 에이전트별 마지막 수신 시간, 오늘 건수 |
| **Pricing** | 가격 동기화 | LiteLLM 동기화 상태, 수동 동기화 버튼 |
| | 모델 단가 | pricing_model 테이블 표시/편집 |
| **Setup** | 설정 가이드 | 에이전트별(Claude/Codex/Gemini) OTel 설정 가이드 |
| **Data** | 내보내기 | CSV/JSON 다운로드 |
| | 정리 | 기간별/에이전트별 데이터 삭제 |
| | DB 통계 | 파일 크기, 테이블별 레코드 수 |

> **기존 Config 탭 → 완전 해체**:
> - FileViewer(편집 기능) → **제거**. Argus는 모니터링 도구이며 편집은 IDE의 역할
> - 변경 이력 + 전후 비교 → **Usage > Impact 탭**으로 이동. 프로젝트/사용자 스코프 구분 추가

| Bottom Bar | h-8, 에이전트별 수집 상태 LED + 총비용 + 활성 세션 상세 (에이전트, 모델, 비용, 최대 3개 표시, 30초 폴링) |
| TopBar 제거 | 전역 필터를 각 페이지 내부로 이동 (Sessions, Usage, Tools만 필터 보유) |
