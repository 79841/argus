# /bugfix — 버그 수정

버그를 체계적으로 수정한다: 재현 → 실패 테스트 → 최소 수정 → 통과 확인.

## 사용법

```
/bugfix <버그 설명 또는 Linear 이슈>
```

## 실행 절차

### 1. 재현 확인

- 버그의 증상과 원인을 파악한다
- 관련 코드를 읽고 문제 지점을 특정한다
- 재현 가능한 조건을 정리한다

### 2. 실패 테스트 작성 (Red)

- 버그를 재현하는 테스트 케이스를 작성한다
- 테스트가 **현재 코드에서 실패하는지** 확인한다

```bash
cd dashboard && pnpm test:run
```

- 기존 테스트 파일이 있으면 그곳에 추가한다
- 없으면 `src/**/__tests__/*.test.ts` 패턴으로 생성한다

### 3. 최소 수정 (Green)

- 버그를 수정하는 **최소한의 변경**만 적용한다
- 관련 없는 리팩토링, 개선, 포매팅 변경을 하지 않는다
- 수정 후 테스트가 통과하는지 확인한다

```bash
cd dashboard && pnpm test:run
```

### 4. 회귀 확인

- 전체 테스트를 실행하여 다른 테스트가 깨지지 않았는지 확인한다
- 수정이 다른 기능에 영향을 주는지 검토한다

### 5. 리뷰 (선택)

- 수정 범위가 크거나 핵심 로직을 변경한 경우 `/review`를 수행한다
- 단순한 오타, 조건문 수정 등은 리뷰를 생략할 수 있다

## 주의사항

- **한 번에 하나의 버그만** 수정한다
- 수정하면서 발견한 다른 문제는 별도 이슈로 등록한다
- 테스트 없이 수정하지 않는다 — 테스트가 회귀를 방지한다
